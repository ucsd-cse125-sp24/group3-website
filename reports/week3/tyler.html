  • Specifically, I fell down a rabbit hole trying to learn how boost::multi_index_container works, because I wanted on the server side to store Sessions in a HashMap that was indexable by both the id of the player and the player’s IP address, because there are different parts of the code where you would want to use one of those values and not the other to index into the sessions.
